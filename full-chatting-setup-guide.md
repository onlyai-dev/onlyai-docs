# Настройка папки «Welcome»

## Что делает папка «Welcome folder»

- **Папка «Welcome folder»** отправляется **новым фанатам** после того, как они подписались на выбранную модель.
- Бот отправляет **все медиафайлы из папки сразу** (всю папку).
- Эти медиафайлы отправляются **вместе с Welcome message**, которое настраивается в Model profile **→ «Welcome flow»**.

✅ Вы можете включить в папку «Welcome folder» **фотографии и видео**.

✅ Вы можете сделать папку «Welcome folder» **бесплатной** или **платной**.

✅ Если она **платная**, вы можете добавить **бесплатное превью** (чтобы фанаты могли посмотреть тизер перед покупкой).

# Настройка папок PPV

## Для чего нужно привязывать папки PPV в OnlyAi Admin

- **Папки PPV** — это библиотека платного контента для бота.
- Вы можете подключить **неограниченное количество** папок PPV.
- Бот использует папки PPV, чтобы **сопоставить запросы фанатов** (потребности, желания, фетиши) с лучшим существующим контентом.

**Очень важно:** чем больше папок PPV вы подключаете, тем **больше контекста** имеет бот — и тем лучше он может выбрать соответствующий PPV.

**Напоминание:** **сначала** необходимо **создать папки на OnlyFans**. В OnlyAi Admin вы можете **подключать** только **существующие** папки OnlyFans (здесь нельзя создавать новые).

✅ Бот отправляет **все медиафайлы из папки PPV сразу** (всю папку).

---

## ❗ Не забывайте обновлять папки PPV (рекомендовано)

**Чтобы бот продолжал точно подбирать контент:**

- Регулярно добавляйте новый контент в папки OnlyFans.
- Если вы создаёте **новую папку** в OnlyFans (особенно для новых постов/медиа), **подключите её также в админке бота**. Это поможет боту точно реагировать на запросы фанатов по медиа-постам.
- Если содержимое папки сильно меняется, обновите её **описание / теги / части тела / интенсивность**, чтобы бот не «неправильно маркировал» её.

# **Content Request (система тикетов)**

## **Что это такое?**

Content Request — это система тикетов, которая:

- фиксирует запросы фанов на конкретный контент
- превращает их в задачи
- позволяет закрывать эти запросы через добавление нужных папок

## **Зачем это нужно?**

У модели есть какое-то количество папок, но они не всегда покрывают все запросы фанов.

Фан может запрашивать конкретный тип контента, которого нет.

Content Request:

- фиксирует реальные запросы
- показывает, какого контента не хватает
- даёт возможность закрыть этот запрос через добавление папки

## **Как это работает**

1. Фан делает запрос в чате
2. Система детектит запрос
3. Создаётся тикет
4. В тикете фиксируется описание запроса

# Настройка VIP Club

VIP Club — это платный доступ для фанов с разовой оплатой (через tip), который даёт доступ к закрытому контенту и дополнительным привилегиям.

После вступления фан получает доступ к выбранной папке и переводится в VIP-сегмент.

## **Как настроить**

1. Перейдите в раздел **Models**
2. Выберите нужную модель
3. Откройте раздел **Settings**
4. Найдите блок **VIP Club**

## **Шаги настройки**

1. Включите **Enable VIP Club**
2. Выберите папку в поле **Folder.** Это папка, к которой получат доступ VIP-фаны
3. Укажите стоимость в поле **Price ($)**Это сумма, которую фан должен отправить через tip для вступления
4. Заполните поле **Description**Укажите, что получает фан:
    - доступ к эксклюзивному контенту
    - приоритет в общении
    - дополнительные бонусы
5. Нажмите **Save VIP Club Settings**

## **Результат**

После настройки VIP Club система готова к работе: фан может вступить через оплату и получить доступ к VIP-контенту, а бот корректно объясняет условия и обрабатывает интерес к клубу в диалоге.

# Настройка Trigger **messages (лайки)**

Bump messages — это функция автоматических сообщений, которые бот отправляет при лайке от фаната, чтобы мягко вернуть пользователя в диалог и повысить вовлечённость.

Функция работает с учётом временных ограничений, чтобы избежать избыточной активности со стороны бота.

💡 Trigger-сообщения срабатывают **только при лайке от фаната**.

Перед отправкой бот проверяет:

- не было ли сообщений от бота за последние **4 часа**
- не было ли отправленных bump-сообщений за последние **8 часов**

Если оба условия соблюдены → бот отправляет trigger-сообщение.

### **Как выбирается сообщение**

- сообщение выбирается **случайным образом из списка добавленных**
- приоритет отдаётся сообщениям, которые ранее не отправлялись
- после использования всех сообщений начинается повторный цикл
